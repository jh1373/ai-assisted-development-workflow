# プロジェクト案内図 UI再設計 計画

## 目的

初心者が数字やファイル名を見て推測しなくても、プロジェクトの全体像、現在の状態、今回見る場所を判断できる画面へ作り直す。

## 設計方針

- ベンチマークの配色やダークテーマを踏襲せず、独自の編集ツール型デザインにする
- 数字は分母、意味、次の行動と一緒に表示する
- ディレクトリ構成は、最上位の構成から詳細ツリーへ段階的に見せる
- 装飾より可読性、情報の階層、操作結果の予測しやすさを優先する
- PC、タブレット、スマートフォンで同じ情報へ到達できるようにする

## 完了条件

- 5つの確認方法と現在位置が明確である
- 役割説明の件数が、良い状態か悪い状態か判断できる
- ルート直下のフォルダと入口をツリーを開く前に理解できる
- 画面幅500px以上で表示や操作が破綻しない
- rootとstarterの画面資産が同期し、全自動テストが成功する
