# localhost tokenログ対策

## 問題

API認証tokenをURLクエリへ含めていたため、PythonのHTTPアクセスログにtokenがそのまま表示されていた。localhost限定かつ閲覧専用でも、画面共有やログ共有による不要な漏えい経路になる。

## 対応

初回受け渡しはHTTPサーバーへ送られないURLフラグメント、API認証は専用HTTPヘッダーへ移行した。加えて、HTTPログ側でも実tokenを`[REDACTED]`へ置換する。

`.cmd`の自動ブラウザ起動では実tokenをブラウザへ直接渡し、ターミナルにはtokenを含まないlocalhost URLだけを表示する。自動起動できない環境では、`--open-browser`なしの明示的な手動起動に限って一時URLを表示する。

## 判断

URLクエリ認証は互換性のために残さず拒否する。ランチャーと画面は同じ導入物として同期されるため、不要な認証経路を残すより安全性を優先した。
