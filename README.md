# AI-assisted Development Workflow

AI開発支援ツールを使う開発で、文脈・計画・検証・判断理由を残すためのワークフローです。

このリポジトリは、AIにコード作成を任せるためのプロンプト集ではありません。  
AIを使うことで失われやすい「なぜその実装にしたのか」「何を試して失敗したのか」「次に何を確認すべきか」を、Gitとドキュメントで追跡できる形にするためのテンプレート集です。

## 解決したい課題

AI開発支援ツールを使うと、実装の速度は上がります。一方で、開発の過程がチャット内に閉じやすく、後から追いにくい状態になりがちです。

特に次のような問題が起きます。

- チャットが長くなり、前提や判断理由が埋もれる
- Gitには差分は残るが、判断理由や失敗した試行は残りにくい
- 別のチャットで再開したとき、プロジェクトの現在地を復元しにくい
- AIの出力が正しいかどうかを、どの観点で確認したかが残らない
- 「完了したつもり」になり、テストやレビューが抜ける

このワークフローでは、1つのタスクを1つの開発セッションとして扱い、作業開始時と終了時に決まった手順を通します。

## 基本方針

```text
要件定義
→ ロードマップ作成
→ フェーズ分割
→ セッション開始
→ 詳細計画
→ 実装
→ 完了前レビュー
→ セッション終了
→ Git commit + devlog
→ 次のセッションへ
```

重要なのは、AIの回答をそのまま記録することではありません。  
開発者が何を前提にし、何を判断し、何を検証したかを残すことです。

## 構成


## 使い方

### 1. 最初に要件とロードマップを作る

開発を始める前に、いきなり実装へ進まず、次の情報を整理します。

- 何を作るのか
- 誰が使うのか
- どの問題を解決するのか
- 完了条件は何か
- どの順番で作るのか
- 今回は何をやらないのか

この段階で、`templates/project-status.md` やプロジェクトごとのロードマップを作成します。

### 2. タスクごとに新しいセッションを始める

1つのチャットに複数の大きな作業を詰め込まず、タスク単位でセッションを分けます。  
新しいセッションでは、`workflows/session-start.md` を使って現在地を復元します。

確認するものの例:

- Gitの作業状態
- プロジェクトルール
- 現在のロードマップ
- 最新の作業ログ
- 今回の対象フェーズ
- テストの状態

### 3. 実装前に計画を作る

実装前に、`templates/implementation-plan.md` を使って計画を作ります。

計画では、次の点を明確にします。

- 今回の目的
- 変更対象
- 実装手順
- 影響範囲
- 検証方法
- 失敗しやすい点
- 今回やらないこと

### 4. 実装後に完了前レビューを行う

実装が終わったら、すぐに完了扱いにせず、漏れを確認します。

確認する観点:

- 計画したタスクがすべて完了しているか
- 余計な変更が混ざっていないか
- テストやビルドは通っているか
- 仕様と実装がずれていないか
- 次の作業者が見て状況を理解できるか

### 5. セッション終了時にdevlogを残す

`workflows/session-end.md` と `templates/devlog.md` を使い、作業ログを残します。

devlogには、単なる作業内容ではなく、判断理由を残します。

- なぜその対応をしたか
- どの選択肢を見送ったか
- どんな問題があったか
- 何を試したか
- 残っているリスクは何か
- 次に何をするべきか

Gitは「何を変えたか」を残します。  
devlogは「なぜ変えたか」を残します。

## このリポジトリが目指すもの

このワークフローは、AI開発支援ツールを使った開発を、再現可能で追跡しやすい形にすることを目的としています。

目指しているのは、AIに任せきりの開発ではありません。  
人間が要件、判断、検証、記録を管理し、AIを実装や調査の補助として使う開発です。

## 対象

- AI開発支援ツールを使って個人開発を進めている人
- 長いチャットで前提が崩れやすいと感じている人
- Gitの差分だけでは判断理由が残らないと感じている人
- devlogや作業ログを残す開発フローを整えたい人
- チーム開発に近い形で個人開発を管理したい人

## 注意

このリポジトリのテンプレートは、そのまま使うよりも、各プロジェクトに合わせて調整することを想定しています。

また、devlogやプロジェクト文書には、秘密情報、認証情報、個人情報、管理画面URL、社内固有情報を含めないでください。
