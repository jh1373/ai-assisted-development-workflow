# Security and Privacy Notes

devlogやプロジェクト文書は、作業の再現性を高めるために残します。
ただし、ログを残すことと、何でも記録することは違います。

## 書いてはいけないもの

- APIキー
- パスワード
- トークン
- 秘密鍵
- Cookie
- 認証ヘッダー
- 管理画面URL
- 個人情報
- 顧客情報
- 社内固有の機密情報
- 外部APIレスポンス全文
- チャット全文

## 書いてよい形

具体的な秘密情報ではなく、概要と影響を書きます。

例:

```text
NG: 管理画面のURLは https://...
OK: 管理画面URLは公開資料に含めない。検証はローカル環境で実施。
```

```text
NG: APIレスポンス全文を貼る
OK: 必要なフィールドだけを匿名化して要約する
```

## 公開前チェック

公開前に、次の観点で確認します。

- 秘密情報が含まれていないか
- 個人名や社名が不要に出ていないか
- 内部URLや管理画面名が出ていないか
- エラーログにトークンやパスが含まれていないか
- 画像やスクリーンショットに個人情報が映っていないか

## Strict modeで追加確認すること

認証、権限、課金、個人情報、データ移行、本番設定を扱う場合は、通常の公開前チェックだけでは足りません。

追加で確認します。

- 誰が何にアクセスできるかが変わるか
- ログに秘密情報や個人情報が出る可能性があるか
- 外部サービスに送るデータが増えるか
- データ削除、移行、復旧不能な変更があるか
- 依存ライブラリや外部APIを追加しているか
- ロールバックできるか
- ロールバックできない場合、そのリスクを人間が承認しているか

詳細は [strict-mode.md](strict-mode.md) を参照してください。

チームでの承認責任やPR運用は [team-development.md](team-development.md) を参照してください。
